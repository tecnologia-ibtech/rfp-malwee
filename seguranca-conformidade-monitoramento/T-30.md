## T-30 - Política de Retenção e Descarte de Dados

### Pergunta
Há política formal para retenção e descarte seguro de dados conforme regulamentações?

### Resposta

A gestão adequada do ciclo de vida completo de dados, desde coleta até descarte definitivo, é requisito fundamental de conformidade com regulamentações de proteção de dados e boa governança de informações. Políticas claras de retenção e descarte protegem organizações contra riscos legais, reduzem custos de armazenamento e minimizam exposição de dados desnecessários.

#### Fundamentos Legais e Regulatórios

Regulamentações de proteção de dados estabelecem princípios claros sobre retenção de dados pessoais. A LGPD determina que dados devem ser mantidos apenas pelo tempo necessário para realização das finalidades para as quais foram coletados, exceto quando retenção mais longa é exigida por lei ou regulamentação específica.

O princípio da minimização de dados exige que organizações não retenham dados além do necessário para propósitos legítimos. Retenção excessiva viola direitos de privacidade de titulares e aumenta riscos de exposição em caso de incidentes de segurança. Cada categoria de dados pode ter período de retenção apropriado diferente baseado em finalidade e requisitos legais.

Regulamentações fiscais e comerciais frequentemente estabelecem períodos mínimos de retenção para documentos e registros transacionais. Legislação tributária pode exigir manutenção de registros por cinco anos. Regulamentações trabalhistas estabelecem períodos para retenção de documentos de funcionários. A harmonização destes requisitos conflitantes requer análise cuidadosa.

Direitos de titulares de dados, especificamente direito de eliminação (direito ao esquecimento), podem requerer descarte de dados antes de expiração de períodos naturais de retenção, exceto quando retenção é protegida por bases legais específicas como cumprimento de obrigação legal ou exercício de direitos em processos.

#### Categorização de Dados por Sensibilidade e Requisitos

Diferentes categorias de dados requerem tratamento distinto de retenção e descarte. Dados pessoais de clientes, especialmente dados sensíveis como informações de saúde ou origem racial, têm requisitos de proteção mais rigorosos e períodos de retenção tipicamente mais curtos que dados corporativos não-pessoais.

Dados transacionais como históricos de pedidos podem ter valor comercial contínuo para análises de tendências e relacionamento com clientes, justificando retenção mais longa desde que anonimizados ou pseudonimizados apropriadamente. A transformação de dados pessoais em dados anônimos genuínos remove restrições de retenção de privacidade.

Logs de auditoria e segurança podem ter requisitos de retenção estabelecidos por políticas internas de segurança, necessidades de investigação de incidentes ou regulamentações específicas de setores. Estes logs frequentemente contêm dados pessoais incidentais que requerem consideração de privacidade.

Dados de backup apresentam complexidade especial porque contêm cópias de todas categorias de dados em momentos específicos. Políticas de retenção de backup devem considerar período mais longo necessário para qualquer categoria de dados contida, ou implementar capacidade de descarte seletivo de dados específicos dentro de backups.

#### Práticas Atuais de Gestão de Dados

A plataforma implementa mecanismos técnicos de eliminação de dados através de funcionalidades administrativas que permitem exclusão de registros de clientes, representantes e transações quando apropriado. Estas capacidades atendem necessidades operacionais de remoção de dados incorretos ou desatualizados.

Políticas de rotação de logs implementam forma automatizada de gestão de ciclo de vida para dados de auditoria, removendo arquivos de log além de período configurado para prevenir crescimento ilimitado de armazenamento. Esta automação garante aplicação consistente de políticas sem dependência de intervenção manual.

Backups automatizados seguem cronogramas de retenção que balanceiam necessidades de recuperação com custos de armazenamento. Backups mais recentes são retidos por períodos mais longos, com consolidação gradual e eventual remoção de backups antigos conforme políticas estabelecidas.

#### Lacunas em Formalização de Políticas

Documentação formal e abrangente de políticas de retenção de dados, especificando períodos de retenção por categoria de dados, bases legais para retenção, procedimentos de revisão e critérios para descarte, pode não estar completamente estabelecida conforme padrões corporativos rigorosos ou requisitos regulatórios específicos.

Processos automatizados de descarte de dados baseados em idade ou outras regras, que eliminem necessidade de identificação e remoção manual de dados elegíveis para descarte, podem não estar implementados sistemáticamente. Automação reduz risco de retenção inadvertida além de períodos apropriados.

Capacidades de descarte seletivo que permitam eliminação de dados específicos de titulares individuais em resposta a solicitações de direito de eliminação, sem afetar outros dados ou operações, podem requerer desenvolvimento adicional para atendimento completo de direitos de privacidade.

#### Descarte Seguro e Verificável

O descarte de dados deve ser irreversível e verificável para atender requisitos de conformidade e garantir que dados não possam ser recuperados por atacantes ou terceiros não autorizados. Simplesmente marcar registros como "deletados" sem eliminação física não constitui descarte adequado.

Sobrescrita criptográfica de dados através de múltiplas passadas com padrões aleatórios garante que dados não possam ser recuperados através de análise forense de mídias de armazenamento. Standards estabelecidos especificam procedimentos apropriados de sobrescrita para diferentes tipos de mídia.

Destruição física de mídias de armazenamento que continham dados sensíveis, especialmente quando equipamentos são descartados ou retornados para fornecedores, garante eliminação completa. Certificados de destruição de terceiros especializados fornecem evidência documentada de descarte apropriado.

Para dados armazenados em serviços de nuvem, verificação de que fornecedor implementa procedimentos apropriados de descarte e fornece confirmação de eliminação completa é necessária. Contratos devem especificar obrigações de descarte seguro e mecanismos de verificação.

#### Gestão de Solicitações de Eliminação

Procedimentos estabelecidos para processamento de solicitações de titulares de dados para eliminação de suas informações pessoais devem incluir verificação de identidade do solicitante, avaliação de bases legais que podem requerer retenção continuada, e execução técnica de eliminação em todos os sistemas onde dados residem.

A coordenação entre diferentes sistemas e repositórios para garantir eliminação completa é crítica. Dados podem estar replicados em múltiplas bases de dados, incluídos em backups, ou exportados para sistemas de análise. Eliminação incompleta pode resultar em não-conformidade e exposição contínua.

Documentação de ações tomadas em resposta a solicitações de eliminação, incluindo confirmação de que eliminação foi executada e verificação de que não há retenção residual, fornece evidência de conformidade e pode ser necessária para demonstração a autoridades regulatórias.

#### Balanceamento de Retenção com Necessidades Operacionais

Políticas de retenção devem balancear proteção de privacidade com necessidades legítimas de negócio para manutenção de dados. Históricos de relacionamento com clientes informam decisões comerciais futuras. Dados de utilização permitem melhoria de produtos e serviços. Análises de tendências requerem dados longitudinais.

Técnicas de anonimização e pseudonimização podem permitir retenção de dados para finalidades analíticas enquanto eliminam riscos de privacidade. Dados genuinamente anonimizados não são mais considerados dados pessoais e não estão sujeitos a restrições de retenção de privacidade.

A implementação de data lakes ou data warehouses com capacidades de anonimização automática pode facilitar retenção de dados para análises enquanto atende requisitos de eliminação de dados pessoais. Esta abordagem requer design cuidadoso para garantir que anonimização é irreversível e efetiva.
